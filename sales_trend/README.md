# sales_trend — 売上推移ダッシュボード

財務部の売上(収入ベース)を積み上げ棒グラフで可視化するポータルアプリ。
CoWork(2026-06-09)で作成した `Qtr別売上推移_基準版.xlsx` を再実装したもの。

## 画面

`/sales-trend/`（ログイン必須）。**サブナビ3構成 × 期間トグル**。単位 **100万元**。

| サブナビ | 含む系列 | = CoWork シート |
|----------|----------|------------------|
| 全体売上 | Faurecia + 大口 + その他 | ①全体_構成別 |
| F社除く全体 | 大口 + その他 | ②F社除く |
| 上海通運移管分込み | 大口 + その他 + グレー(=上海通運移管分) | ③F社除く・グレー含む |
| 顧客別 | Faurecia + 大口顧客 各社 + その他(残り全部まとめ) | — |

- 期間トグルで **四半期 / 月別** を切替（月別は全月 2022/01〜最新）。
- 各積み上げ棒の上に **合計(100万元)を縦書き**で表示。
- 単位は**グラフ左上**、凡例は**上・右寄せ横並び**。サブナビ切替は**アニメーション**で変化。
- 系列(上→下): Faurecia / 大口顧客 / その他22以前 / 23 / 24 / 25 / 26新規 / グレー(最下段)。
  色=意味: 青系=大口アンカー、緑系=「その他」コホート(新しいほど淡)、グレー=調整。
- **縦軸は min0 固定・max を3構成で共通化**(四半期/月別ごと)。3構成はクライアント側切替
  (リロードなし)＋位置・サイズ・縦軸を揃え、切替時にチカチカしない。
- 日中両言語対応(テンプレート・系列ラベル・合計/ツールチップ)。

## データの流れ

```
sales.db (財務部売上, Mac の NGLS-DB)
   │  manage.py build_seed   ← 分類を適用して seed.json を再生成(開発機で実行)
   ▼
sales_trend/data/seed.json   ← コミット対象
   │  migration 0002_load_seed (冪等)
   ▼
本番 db.sqlite3 の Customer / MonthlySale
   │  views.data_api が四半期/月別×全系列を集計
   ▼
Chart.js (app.js) がトグルで描画
```

本番(Windows)は `sales.db` を持たないため、データ更新は **Mac で `build_seed` →
seed.json をコミット → 本番 `migrate`** の手順。`only_if_empty=True` なので既存環境では
再投入されない（入れ替える場合は手動 truncate か migration 追加が必要）。

## 分類ルール（build_seed / 引き継ぎ §7 準拠）

- `category=Faurecia` → faurecia
- `category=大口顧客` → major
- `category=調整` または グレー固定ID該当 → grey
- 上記以外 → other。**開始年は「実際に初めて売上が立った年」**で判定し、22年以前は 2022 に丸める。

グレー固定ID・グループ寄せ等の元ルールは
`/Users/toshmurayama/NGLS-DB/引き継ぎ_2026-06-09_CoWork作業.md` §7 を参照。

## 既知の差異（CoWork xlsx との照合）

- **合計・主要系列(Faurecia/大口)・2022Q1 は xlsx と完全一致**。各期の総額も一致。
- 末尾の数顧客で「グレー所属」「その他の開始年(22以前↔23)」が CoWork と異なる
  （例: 2025/01 の その他22以前/23/グレー の内訳）。CoWork は当時アドホックに分類しており、
  個別の顧客→バケツ対応が引き継ぎに残っていないため。**集計総額には影響しない**ため
  本実装の(コード化され再現可能な)分類を正とする。再調整が必要なら build_seed の
  `GREY_IDS` / `classify()` を編集する。
