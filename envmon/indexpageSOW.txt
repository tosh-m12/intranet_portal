📘 index.html 仕様書（温湿度モニタリング・トップ画面）

最終更新：2025-12-XX
対象：templates/envmon/index.html
目的：倉庫別の温湿度リアルタイム表示および7日間履歴の可視化

1. 画面概要

本画面では、倉庫別に以下の情報を表示する。

リアルタイム温度・湿度の平均値（タコメーター風ゲージ）

直近 7 日間の温湿度推移（3 時間ごとのスロットで構成された折れ線グラフ）

倉庫単位で 1 セットのブロックを構成し、画面には倉庫数分が 縦に並んで表示される。

2. データ取得仕様
2.1 リアルタイム値（温度・湿度）

使用 API：/envmon/data_api

外部サービス（1weilian）のリアルタイムデータを取得して返す

画面側の更新間隔：EnvSettings.interval 秒

倉庫に複数デバイスがある場合は 温度・湿度の平均値 を算出して表示

未割当デバイス、未登録デバイス（名前が [未登録: で始まる）は除外

2.2 7 日間グラフ用データ

使用 API：新規作成 /envmon/history_7days

データソース：DeviceHistory テーブル

表示対象期間：直近 7 日間（168 時間）

スロット：1 日を 3 時間単位で区切る
00:00, 03:00, 06:00, 09:00, …, 21:00 の計 8 スロット/日

倉庫のデバイスが複数ある場合：

スロット内で 最初に記録されたデータのみ使用（平均値を取らない）

ただし複数デバイス分をまとめるときは 平均値を算出して 1 本の線にする

戻り値は倉庫コードごとに：

timestamps（スロット時刻の配列）

temperature_values（平均値）

humidity_values（平均値）

3. 画面レイアウト仕様
3.1 倉庫ブロックの縦並び構造

画面には次のように、倉庫ごとのブロックを縦方向に並べる。

─────────────
■ 倉庫A
   ├─ 左：7日間グラフ
   └─ 右：温度計（ゲージ）＋湿度計（ゲージ）
─────────────
■ 倉庫B
   ├─ 左：7日間グラフ
   └─ 右：温度計・湿度計
─────────────
■ 倉庫C
   ...

3.2 倉庫ブロックの HTML 構造

JS により倉庫ごとに次の要素を生成する：

<div class="warehouse-block" data-location-id="locXX">
    <h2 class="warehouse-title">倉庫名</h2>

    <div class="warehouse-main">

        <!-- 左：7日間グラフ -->
        <div class="warehouse-graph">
            <canvas id="chart-locXX"></canvas>
        </div>

        <!-- 右：タコメーター風温湿度ゲージ -->
        <div class="warehouse-gauges">
            <div class="gauge gauge-temp" id="gauge-temp-locXX"></div>
            <div class="gauge gauge-hum"  id="gauge-hum-locXX"></div>
        </div>

    </div>
</div>

4. タコメーター風温度計・湿度計のデザイン仕様

以下のデザインを基準とする（参考画像あり）：

4.1 外観（タコメーター風）

黒背景（円形の内部は黒〜ダークグレー）

外周リングにカラーグラデーションの帯

低温（低湿）側：ブルー

適正範囲：グリーン

高温（高湿）側：オレンジ〜レッド

光沢感を出すための radial-gradient（上部を少し明るく）

4.2 目盛り

白色の目盛り線を外周に沿って配置

5℃ / 10％間隔で数字を表示

数字は外周の内側に配置

4.3 針の仕様

細い赤いライン

中心から外側に伸びる

根本に赤い丸

値に応じて CSS transform: rotate() で回転させる

4.4 単位・タイトル

メーター内に以下の文字を表示：

温度計：Temperature / °C

湿度計：Humidity / %RH

4.5 設置位置

温度計と湿度計は横並びに配置する：

[ 温度計 ] [ 湿度計 ]

5. グラフ仕様（左側）

使用ライブラリ：Chart.js

表示：折れ線グラフ（温度と湿度の二系統）

X 軸：7 日間 × 8 スロット（3 時間ごと）

Y 軸：自動スケール

スタイル：

温度線：赤系

湿度線：青系

6. JS 動作仕様（envmon_index.js）
6.1 初期ロード

/envmon/data_api を呼び出す

デバイスを倉庫別に分類してリアルタイム値を計算

倉庫数分の HTML ブロックを生成

タコメーターゲージを初期化

6.2 グラフ取得

/envmon/history_7days を呼び出す

返ってきたデータを倉庫ごとに Chart.js へ描画

6.3 自動更新

interval 秒ごとにリアルタイム部分（ゲージ）を更新

グラフの更新は原則 1 回（ページロード時のみ）

7. index.html 内の必須要素
<div id="envmon-root"
     data-api-url="{% url 'envmon:data_api' %}"
     data-history-url="{% url 'envmon:history_7days' %}"
     data-interval="{{ interval }}">
</div>

<div id="warehouse-sections"></div>

<script src="{% static 'envmon/envmon_index.js' %}"></script>

8. 非表示対象

以下のデバイスは倉庫表示に含めない：

未割当デバイス

[未登録: で始まる名称のデバイス
